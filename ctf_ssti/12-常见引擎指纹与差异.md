# 常见引擎指纹与差异

## 这篇的目标
- 不是让你精确识别所有版本。
- 而是让你在 CTF 里用最少测试快速缩小范围。

## 一组基础指纹

### 测试组 A：数学表达式
```txt
{{7*7}}
${7*7}
#{7*7}
<%= 7*7 %>
```

### 测试组 B：字符串能力
```txt
{{"a"*3}}
${"a"?upper_case}
```

### 测试组 C：报错收集
- 故意访问不存在对象
- 故意闭合不完整语法
- 观察栈信息、类名、模板文件后缀

## 常见判断经验

### 偏 Jinja2 / Twig
- `{{7*7}}` 返回 `49`
- 报错出现 Jinja2 或 Twig 相关类名
- 页面模板本身常见花括号表达式

### 偏 Freemarker
- `${7*7}` 生效
- 报错里出现 `freemarker`
- 模板文件常见 `.ftl`

### 偏 Velocity
- `$var`、`#set` 风格明显
- 报错里出现 `velocity`

### 偏 EJS
- `<%= 7*7 %>` 生效
- Node 栈报错或模板后缀提示

## Jinja2 与 Twig 的区别提醒
- 语法表面相像，但对象生态不同。
- 看见 `{{7*7}}` 成功，不代表后续 Jinja2 对象链一定适用。
- 要继续用对象名和报错信息确认所属栈。

## 观察模板文件后缀
- `.html` 不说明问题
- `.j2`、`.jinja2`、`.twig`、`.tpl`、`.ftl`、`.vm` 很有帮助

## 观察报错关键词
- `jinja2.exceptions`
- `Twig\Error`
- `Smarty`
- `freemarker.template`
- `org.apache.velocity`

## 初学者最低要求
- 不要求一眼识别版本。
- 但至少要能把 Python/PHP/Java/Node 的大方向区分开。
