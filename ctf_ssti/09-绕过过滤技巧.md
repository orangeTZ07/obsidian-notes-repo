# 绕过过滤技巧

## 常见过滤对象
- 特殊字符：`.`, `_`, `[`, `]`, `(`, `)`, `'`, `"`
- 关键字：`class`, `mro`, `subclasses`, `globals`, `builtins`, `import`, `os`
- 长度限制：限制 payload 总长度
- 黑名单正则：匹配特定敏感单词

## 绕过思路不是“硬背”，而是“换等价访问方式”

### 思路 1：点号访问换成属性函数
- 原本：`obj.attr`
- 替代：`obj|attr("attr")`

### 思路 2：属性名拆分拼接
- 原本：`__class__`
- 替代：通过字符串拼接、格式化、变量中转构造

### 思路 3：索引访问与属性访问互换
- 原本：`obj["x"]`
- 替代：`obj.x`
- 或反过来，用字典索引规避关键字过滤

### 思路 4：利用已有变量中转
- 先把一段字符串存在模板变量里
- 再通过拼接、过滤器或下游语句继续利用

### 思路 5：少字符打法
- 题目只给很短输入时，优先测已有对象名和短表达式
- 想办法借模板上下文中已有变量名，而不是自己从零构造全链

## Jinja2 常见替代技巧

### `attr` 过滤器
```jinja2
{{ request|attr("args") }}
```

### 字符串拼接
```jinja2
{{ "ab" ~ "cd" }}
```

### 先取短对象，再继续延展
```jinja2
{{ config }}
{{ self }}
{{ url_for }}
```

## 过滤器绕过的实战习惯
- 先判断是前端过滤还是后端过滤。
- 先判断是直接删除、替换为空，还是触发拦截。
- 不同处理方式决定你该用“编码”“拆分”还是“换语法”。

## 黑名单题的解题顺序
1. 先拿最短可执行表达式
2. 统计哪些字符和单词会触发拦截
3. 找等价访问路径
4. 缩短 payload，避免一次塞太多东西
5. 成功后记录可用字符集，方便继续扩展

## 容易被忽略的绕过点
- 请求参数可能在第二次渲染时才触发
- URL 编码、双编码对后端框架解析结果不同
- 某些题会过滤输出，但不影响内部执行

## 经验结论
- 真正稳定的绕过来自“理解对象访问语义”，不是抄一条万能字符串。
- 如果你连当前表达式在做什么都解释不清，就不算掌握这个绕过。
