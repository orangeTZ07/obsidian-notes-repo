# Sandbox 逃逸与对象利用

## 什么是沙箱
- 模板引擎为了限制危险能力，可能禁止某些属性、方法、导入或执行器。
- 但沙箱不等于绝对安全，只要还能访问足够多的对象，就有机会绕过去。

## 你应该怎么理解“逃逸”
- 不是神秘技巧。
- 本质是：在受限语义里，继续找到一个能通往危险能力的对象路径。

## Jinja2 中常见的逃逸关注点
- `__globals__`
- `__builtins__`
- 上下文里的函数对象
- 可枚举到的类和子类
- 模板内部辅助对象，如 `cycler`、`joiner`、`namespace`

## 为什么函数对象危险
- Python 函数常带 `__globals__`
- `__globals__` 里可能有：
  - `__builtins__`
  - 已导入模块
  - 当前应用对象

只要能进入这层，后面就经常能继续走到文件操作或系统命令。

## 为什么类层级危险
- 很多对象都能拿到 `__class__`
- 类可以继续到继承链，再到 `__subclasses__()`
- 某些子类对应文件类、异常类、进程类、导入辅助类，能成为跳板

## 逃逸题做法
1. 先证明当前对象访问能力存在
2. 再找没有被过滤的危险入口
3. 优先选链短、可解释、稳定的路径
4. 如果子类索引不稳定，先枚举再筛选，不要盲信固定编号

## 常见误区
- 误区 1：沙箱逃逸就是背某个 `__subclasses__()[xxx]`
  - 纠正：那只是某个环境下的结果，不具普适性。
- 误区 2：只要拦了 `__` 就没法继续
  - 纠正：很多题是通过别的对象入口或字符串拼接继续走。
- 误区 3：拿到命令执行才算成功
  - 纠正：很多题只要读到 `flag` 就够了，没必要额外扩大动作面。

## 出题人常放的坑
- 把危险关键字过滤得很明显，诱导你只试死链
- 让你必须先通过 `config`、`request`、`self` 等低调对象找到真正入口
- 返回结果被截断，导致你以为链无效，其实只是输出没显示全

## 初学者建议
- 先学会解释“这条链为什么能走通”，再去收集更多链。
- 每成功打通一次，就把“入口对象 -> 中间桥 -> 危险能力”记成三段式。
