# PHP 与 Java 模板引擎速览

## 为什么要学这一篇
- 很多初学者一看到模板注入就只会打 Jinja2。
- CTF 出题人经常利用这种惯性，让你在 PHP 或 Java 栈上踩坑。
- 这一篇不追求把每个引擎打到极致，而是帮你建立“看见语法就能猜方向”的能力。

## Twig（PHP）

### 常见语法
```twig
{{ 7*7 }}
{{ app.request }}
```

### 识别信号
- 报错里可能出现 `Twig\Environment`、`Twig\Template`
- 页面模板语法和 Jinja2 长得很像，但对象和扩展机制不同

### 做题重点
- 先确认是否允许表达式求值
- 再看能否访问 `app`、`_self` 等对象
- 某些版本或配置下可能存在危险过滤器、函数或模板 include 相关问题

## Smarty（PHP）

### 常见语法
```smarty
{$name}
{if 1}ok{/if}
```

### 识别信号
- 报错中出现 `Smarty`
- 花括号结构明显，但并不总支持你熟悉的 Jinja2 风格表达式

### 做题重点
- 注意 `{php}`、`string:`、注册修饰符、模板资源类型等出题点
- 很多题目会让你从“变量输出”升级到“模板资源滥用”

## Freemarker（Java）

### 常见语法
```ftl
${7*7}
${"abc"?upper_case}
```

### 识别信号
- 报错里出现 `freemarker`
- 经常见于 Java Web、CMS、OA、报表系统题目

### 做题重点
- 关注内建函数、对象包装器、模板工具类
- 典型题会围绕类访问、对象包装、执行器对象展开

## Velocity（Java）

### 常见语法
```velocity
#set($x = 7*7)
$x
```

### 识别信号
- 报错或类名里出现 `velocity`
- 模板里大量使用 `$var`、`#set`、`#foreach`

### 做题重点
- 注意上下文工具对象
- 注意反射相关利用链和类访问能力

## Node 模板引擎
- EJS 常见语法：`<%= name %>`
- Pug、Handlebars、Nunjucks 也可能出题
- CTF 里相对少一些，但只要看到 `<% %>` 或类似表达式，也要纳入 SSTI 检测范围

## 实战建议
- 先识别引擎，再上 payload。
- 报错页面往往是最省时间的情报来源。
- 不同引擎的“高危对象”不同，不要把 Python 链强行搬到 PHP/Java 题上。
