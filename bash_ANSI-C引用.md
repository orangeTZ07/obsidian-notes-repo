ANSI-C引用会将字符串中的转义序列按照 C 语言标准进行解释
```bash
# 八进制转义可以省略前导0
$'\154'        # 3位：\154
$'\54'         # 2位：\54 -> 也是 'l'（八进制54=十进制44='\,'？不对！）

# 注意：八进制数字范围 0-377（对应 ASCII 0-255）
$'\400'        # 错误：超出范围（八进制400=十进制256）
```
注意事项：
```bash
#1. **Shell 必须是 Bash**（或 Zsh，但不是 Dash/Sh）或者别的支持ANSI-C引用的shell
# ✅ 正确
$'\154\163'

# ❌ 错误（双引号不行）
$"\154\163"    # 不会解析转义

# ❌ 错误（没有引号不行）
$\154\163      # 语法错误

#$会把后面引号内的内容整个解析成一个命令
$'\154\163\40\55\154\141' 
#>ls -la: command not found
```

| 转义序列         | 含义               | 示例                 |
| ------------ | ---------------- | ------------------ |
| `\n`         | 换行符 (LF)         | `$'a\nb'`          |
| `\t`         | 制表符 (TAB)        | `$'a\tb'`          |
| `\r`         | 回车符 (CR)         | `$'a\rb'`          |
| `\\`         | 反斜杠              | `$'\\'`            |
| `\'`         | 单引号              | `$'\''`            |
| `\"`         | 双引号              | `$'\"'`            |
| `\0nnn`      | 八进制字节（1-3位数字）    | `$'\012'` 等同于 `\n` |
| `\xHH`       | 十六进制字节（1-2位）     | `$'\x0A'` 等同于 `\n` |
| `\uHHHH`     | Unicode (4位十六进制) | `$'\u4E2D'` = '中'  |
| `\UHHHHHHHH` | Unicode (8位十六进制) | 用于 emoji 等         |
