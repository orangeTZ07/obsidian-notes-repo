# 🌳 二叉树核心概念辨析卡片

期末复习时，我们经常会被这三个长得很像的名字搞混。其实把它们“脱去外衣”看本质，非常简单！

## 1. 二叉搜索树 (BST) 与 二叉排序树
**结论：它们是完全一样的东西！仅仅是名字不同！**就像“西红柿”和“番茄”。

- **英文学名**：Binary Search Tree (BST)
- **为什么叫二叉“搜索”树？**
  因为它的特性（左小、右大）天生就是为了**快速查找（搜索）**而生的，就像查字典一样。
- **为什么严蔚敏版教材叫它二叉“排序”树？**
  因为如果你对这棵树进行**中序遍历**（左->根->右），打印出来的数字一定是一个**从小到大排好序**的序列。

**🌟 考点提醒**：
二叉排序树的**平均**查找时间复杂度是 $O(\log n)$（第16题考点）。但如果不加限制，它最坏可能退化成一根“单链表”（比如依次插入 1, 2, 3, 4, 5），这时候查找的最坏时间复杂度是 $O(n)$。

---

## 2. 判定树 (Decision Tree)
**结论：判定树是一棵“虚拟的树”，它不是用来存数据的，而是用来“分析算法过程”的工具！**

- **本质**：记录你在做算法（比如二分查找）时，大脑里每一次“判断”的轨迹。
- **直观例子**：对有序数组 `[10, 20, 30, 40, 50, 60, 70]` 做二分查找。
  1. 第一次判断：取中间元素 `40` （这就是树根）
  2. 如果要找的数比 `40` 小，你就去左半边 `[10, 20, 30]` 里找中间元素 `20`（这就是左孩子）
  3. 如果比 `40` 大，去右半边找 `60`（这就是右孩子）

画成判定树就是：
```text
      40
    /    \
  20      60
 /  \    /  \
10  30  50  70
```
**🌟 考点提醒**：
二分查找的判定树，一定是一棵**平衡的二叉树**。所以它的高度永远贴近 $\log_2 n$。这就直观地解释了为什么第18题，二分查找的时间复杂度是 $O(\log n)$！
