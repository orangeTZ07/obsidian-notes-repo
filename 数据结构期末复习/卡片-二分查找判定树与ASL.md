# 🌲 判定树 (Decision Tree) 专项笔记

> 判定树是用来**分析算法过程**的思维工具，最常用于分析“二分查找”的性能。

## 1. 判定树长什么样？
假设我们有一个有序数组（长度 $n=11$）：`[5, 13, 19, 21, 37, 56, 64, 75, 80, 88, 92]`

二分查找每次都取中间值 `mid = (low + high) / 2` （向下取整）。
把查找的过程画出来，就是一棵判定树：

```text
                  56(下标5)
               /            \
          19(下标2)          80(下标8)
         /       \          /       \
      5(0)      37(4)    64(6)      92(10)
        \       /          \        /
      13(1)   21(3)       75(7)   88(9)
```

## 2. 判定树的 3 个重要性质
1. **树的形态完全固定**：只要数组长度 $n$ 确定，无论数组里的数字是什么，这棵判定树的形状就是**唯一确定**的！
2. **一定是一棵平衡树**：左右子树的节点数最多相差 1。
3. **高度（深度）**：具有 $n$ 个节点的判定树，高度 $h = \lfloor \log_2 n \rfloor + 1$。这也是二分查找最坏情况下的查找次数。

## 3. 如何计算 ASL (平均查找长度)？
**ASL = (每一层节点数 × 层数) 的总和 / 总结点数 n**

以这棵 $n=11$ 的树为例：
- 第 1 层：1 个节点 (56)，查找 1 次
- 第 2 层：2 个节点 (19, 80)，查找 2 次
- 第 3 层：4 个节点 (5, 37, 64, 92)，查找 3 次
- 第 4 层：4 个节点 (13, 21, 75, 88)，查找 4 次

$$ASL_{成功} = \frac{1 \times 1 + 2 \times 2 + 4 \times 3 + 4 \times 4}{11} = \frac{1 + 4 + 12 + 16}{11} = \frac{33}{11} = 3$$
这意味着在这个长度为 11 的数组里用二分查找，平均只要找 3 次就能找到目标！
