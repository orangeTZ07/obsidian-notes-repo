# 折半查找中点防坑指南 (取左还是取右？)

## 坑点描述
当有序表中有**偶数**个元素时，正中间会卡在两个元素之间。此时，折半查找计算中点时，应该选左边那个，还是右边那个？
*(例如：长度为10的数组，中间是选第5个还是第6个？)*

## 严蔚敏教材与C语言铁律
在 C 语言的底层逻辑和严蔚敏版教材中：
`mid = (low + high) / 2`

**C语言整数除法规则：直接截断小数部分（向下取整）。绝对没有四舍五入！**
例如：$(1 + 10) / 2 = 11 / 2 = 5$

👉 **结论：在偶数个元素中取中点，永远取偏左的那一个！**

## 考试秒杀技巧
如果考卷上出现推断“折半查找比较序列”的选择题，如果在某个回合中，数组剩下了偶数个，而某个选项取了**偏右**的中间数（比如 11/2 它算成了 6），**直接排除，这个选项必然是错的！**
