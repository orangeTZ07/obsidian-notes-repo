# 克鲁斯卡尔算法 (Kruskal's Algorithm)

## 📌 核心思想：精打细算的“包工头”
与 Prim 算法从一个点往外扩张不同，Kruskal 算法具有**全局视野**。它不管顶点在哪里，只盯着**所有边的权值**看。

**行动口号**：“全图找最便宜的路修，只要不兜圈子就行！”

## 📝 算法步骤 (手推做题必备)
1. **报价单排序**：将图中所有边按权值从小到大排序。
2. **贪心挑边**：从最小的边开始，依次考察：
    - 如果这条边连接的两个顶点，目前还**不连通**（即加入后**不会形成回路**），则**保留**这条边。
    - 如果加入后会**形成回路**，则**果断舍弃**。
3. **见好就收**：当选够了 **$n-1$** 条边（$n$ 为图的顶点数）时，算法结束，最小生成树构建完成。

## ⚠️ 严蔚敏教材重难点：如何判断“是否形成回路”？
在代码实现中，Kruskal 算法依赖一种非常重要的数据结构来判断连通性：**并查集 (Disjoint Set)**。

*   **直观理解**：相当于“认大哥”机制。
    *   初始时，每个顶点各自为一个集合（自己是自己的大哥）。
    *   每次考察一条边 $(u, v)$，就去查找 $u$ 和 $v$ 各自的集合代表（大哥）。
    *   **如果大哥相同**：说明它们已经在同一个集合里了，连上必然出现回路（丢弃）。
    *   **如果大哥不同**：说明加入不形成回路（选中），然后执行 `Union` 操作，将这两个集合合并。

## ⚔️ Prim vs Kruskal 对比

| 特性 | Prim (普里姆) | Kruskal (克鲁斯卡尔) |
| :--- | :--- | :--- |
| **视角** | 顶点为中心 (步步扩张) | 边为中心 (全局贪心) |
| **做法** | 找已访问集合与未访问集合之间权值最小的边 | 将边排序，从小到大挑不构成回路的边 |
| **适合场景** | 稠密图 (边多点少)，时间复杂度 $O(V^2)$ 或 $O(E \log V)$ | 稀疏图 (边少点多)，时间复杂度 $O(E \log E)$ |
| **依赖结构** | 数组/优先队列 | 排序算法 + 并查集 |
