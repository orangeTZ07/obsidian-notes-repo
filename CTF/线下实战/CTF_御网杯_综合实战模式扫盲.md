# 御网杯：综合实战模式（应急响应 + 组网防御 + 靶场渗透）扫盲

如果不是 AWD，那说明你们这次采用的是**“综合实战靶场模式”（Cyber Range / 闯关模式）**。这其实更贴近真实的政府/企业网络攻防演练！

在这种模式下，你**不需要**和其他队伍互相攻击，而是官方会发给你们一套或多套独立的网络环境。你们扮演的角色会在 **[蓝队 (Blue Team)](https://en.wikipedia.org/wiki/Blue_team_(computer_security))** 和 **[红队 (Red Team)](https://en.wikipedia.org/wiki/Red_team)** 之间切换。

## 1. 核心考核阶段解析
*   **应急响应（扮演蓝队）**：系统已经被“虚拟黑客”攻破了。你需要在一片狼藉的服务器中查日志、找木马、看流量、分析攻击者的入侵路径，并出具报告或提交黑客留下的特定 Flag。
*   **组网与防御（基础设施）**：你会被分配到一台虚拟路由器或防火墙，需要配置 NAT、路由转发、VPN、[DMZ (非军事区)](https://en.wikipedia.org/wiki/DMZ_(computing))，将不同网段连通。如果组网没搞好，你甚至连靶机都连不上。
*   **靶场渗透（扮演红队）**：官方给你一个目标网络（模拟某公司内网），你需要从边界 Web 突破，拿到外网权限后，进行**内网穿透**，横向移动拿下一台台内网核心服务器的域控权限（Domain Admin）。

## 2. 真实考核场景举例（从易到难）

### 例子1（简单）：Web 访问日志溯源（应急响应）
*   **场景**：题目告诉你服务器的数据库刚刚被脱裤（数据泄露）了。
*   **行动**：你直接去拉取 `/var/log/nginx/access.log`。使用 `grep` 或 `awk` 筛选 HTTP 状态码为 200 的请求，发现某个 IP 在高频访问 `login.php`，并且 URL 里面带了一大堆 `%27%20UNION%20SELECT`。
*   **解题**：将其 URL 解码，发现是 SQL 盲注。你将攻击者的真实 IP、注入的时间点、使用的工具（比如 sqlmap 的 UA 特征）提交得分。

### 例子2（复杂）：边界突破与内网代理转发（渗透与组网）
*   **场景**：你们进入了渗透环节，面对一个只开放了 80 端口的目标边界服务器，且这台服务器内网还有其他不出网的主机。
*   **行动**：
    1. 你利用边界服务器上的 ThinkPHP 反序列化漏洞拿到了 WebShell。
    2. 你发现内网还有一台 `192.168.50.10` 的机器，但这台机器你的电脑无法直接访问（因为不在同一网段）。
    3. **内网穿透（组网）**：你在边界服务器上上传了 `FRP` 或 `Chisel`，将其作为代理节点（Socks5 代理）。
    4. 你的本地电脑配置 `Proxychains`，将所有攻击流量通过边界服务器转发进内网，从而使用本地的 Nmap 对 `192.168.50.10` 进行内网扫描。

### 例子3（地狱级）：联动模式——用敌人的武器打敌人（坑点极多）
*   **场景**：赛制将“应急响应”和“靶场渗透”串联。
*   **应急坑点**：你拿到一段 PCAP 流量包和一台被感染的主机内存镜像文件（Dump）。你必须用 `Volatility` 进行内存取证，发现一个被隐藏的恶意进程，并从中提取出一个极其冷门的加密勒索软件木马。同时，你还要在流量包里分析出这个木马通信时的**自定义加密密钥**。
*   **渗透坑点**：到了渗透环节，你发现目标靶场所有的常规漏洞（如 SQLi、RCE）全部被 WAF 拦截。
*   **破局**：你必须利用**应急响应阶段提取出来的那个特定木马**和**加密密钥**，伪装成该木马的控制端（C2服务器），向目标靶场发送特定构造的协议包来触发后门，从而拿下权限。如果你在上一关应急响应没把流量协议分析透彻，你的渗透环节将直接卡死，零分出局！

## 3. 关联知识点拓展
*   **代理转发与内网穿透 (Pivoting & Tunneling)**：如何把原本进不去的内网流量带出来。
*   **内存取证 (Memory Forensics)**：硬盘里删得干干净净的恶意代码，在内存里依然原形毕露。
*   **Windows 域渗透 (Active Directory Exploitation)**：线下靶场渗透的终极目标往往是拿下域控制器 (DC)，这需要掌握 Pass-The-Hash、黄金票据等技术。

## 4. 推荐项目与论文
*   **[FRP](https://github.com/fatedier/frp) / [Chisel](https://github.com/jpillora/chisel)**：必须熟练掌握的内网穿透神器。
*   **[Volatility](https://github.com/volatilityfoundation/volatility3)**：全球最顶级的内存取证框架。
*   **[CyberChef](https://gchq.github.io/CyberChef/)**：被誉为“网络安全瑞士军刀”，在日志分析中快速解码 Base64、URL、Hex、甚至解密 AES 必备。
*   **学术阅读**：查阅**《The Diamond Model of Intrusion Analysis》（入侵分析钻石模型）**，它会教你如何从 攻击者、受害者、基础设施、技术能力 四个维度去写一份完美的高分应急响应溯源报告。

## 5. 随堂自测题（试着回答或思考）
1. **易**：在 Web 日志里，如果你看到大量的 `%2E%2E%2F%2E%2E%2F`，这通常是什么类型的攻击特征？
2. **中**：在使用 FRP 进行内网穿透时，什么是“正向代理”，什么是“反向代理”？如果目标内网机器可以上网，但外部绝对无法 ping 通它，你应该用哪种方式把它的流量带出来？
3. **难（刁钻题）**：在应急响应内存取证时，你使用 Volatility 的 `pslist` 插件看不到某个恶意进程，但是用 `psxview` 插件时，却发现该进程的 `pslist` 列显示为 False，而 `pspcid` 列显示为 True。请问 Rootkit 是通过断开操作系统内核中的哪个数据结构链表来实现这种隐藏的？
