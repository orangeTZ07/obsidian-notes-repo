# 模板引擎与语法基础

## 模板引擎到底解决什么问题
- 把后端数据渲染成 HTML、邮件、配置片段、报表文本。
- 常规用法是：模板作者预先写好结构，然后把变量填进去。
- 一旦把用户输入当成“模板源码的一部分”，渲染器就可能把攻击者输入解释为模板语法。

## 常见模板构成
- 输出表达式
  - 常见形式：`{{ name }}`、`${name}`、`<%= name %>`
- 控制语句
  - 例如 `if`、`for`、`include`
- 过滤器或函数
  - 例如 `{{ name|upper }}`
- 对象访问
  - 例如 `user.name`、`user["name"]`

## 从“安全插值”到“危险拼接”

### 相对安全的方式
```python
return render_template("hello.html", name=user_input)
```

如果模板文件里只是 `{{ name }}`，模板源码是固定的，用户输入只是数据。

### 危险的方式
```python
return render_template_string("Hello " + user_input)
```

这里用户输入进入的是“模板源码”，引擎会重新解析。

## 你需要认识的几类语法

### Jinja2 常见语法
```jinja2
{{ 7*7 }}
{{ config }}
{{ request.args }}
{% if 1 %}ok{% endif %}
```

### Twig 常见语法
```twig
{{ 7*7 }}
{{ app.request }}
```

### Smarty 常见语法
```smarty
{$name}
{if 1}ok{/if}
```

### Freemarker 常见语法
```ftl
${7*7}
${"abc"?upper_case}
```

## 为什么有的表达式不会执行
- 输入最终只是普通文本，没有进入模板解析器。
- 被做了 HTML 转义，但没进入模板语法层。
- 引擎只允许变量替换，不允许复杂表达式。
- 开发者做了上下文隔离或沙箱限制。

## 适合初学者的判断思路
1. 我输入的内容是进入模板“数据槽位”还是“模板源码”？
2. 如果能执行表达式，引擎大概率属于哪种语法体系？
3. 当前可见的输出，是表达式结果、报错信息，还是被原样转义？

## CTF 中最容易出现 SSTI 的业务点
- 昵称、签名、欢迎语预览
- 邮件模板、工单模板、自定义消息模板
- 报错页、调试页、自定义主题
- PDF/报表/导出模板
- 后台 CMS 或低代码系统的自定义渲染组件

## 初学者行动建议
- 先记住“语法外观”，再记“高危链”。
- 不要一上来就背复杂 RCE payload，先学会判断引擎和回显类型。
