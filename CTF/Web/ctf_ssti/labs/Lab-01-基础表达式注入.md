# Lab 01：基础表达式注入

## 练习目标
- 识别模板表达式是否被执行
- 学会从“原样输出 / 报错 / 求值成功”三类结果判断下一步

## 场景描述
- 页面提供一个昵称输入框
- 提交后会在欢迎语中显示：`Hello, <你的输入>`
- 你怀疑后端不是简单拼接，而是把整段欢迎语当模板渲染

## 建议测试顺序
1. 输入普通字符串，记录页面基线
2. 输入 `{{7*7}}`
3. 输入 `${7*7}`
4. 输入 `#{7*7}`
5. 输入一个会触发报错的表达式

## 你应该观察什么
- 是否出现 `49`
- 是否原样显示花括号
- 是否出现 500 错误或模板异常栈
- 响应长度是否变化

## 预期收获
- 能区分“没进模板”和“进了模板但语法不对”
- 能把模板注入的探测流程固定下来

- 你最终凭什么判断这是 SSTI？
- 你是如何缩小引擎范围的？
