# 内网探测方法

## 目标
在 SSRF 场景下，用最少请求确定：
- 哪些主机存活
- 哪些端口开放
- 哪些服务可利用

## 优先级策略（CTF）
1. 本机：`127.0.0.1`
2. 网关与常见内网 IP：`192.168.0.1`、`10.0.0.1`
3. 服务端口：`80`、`8080`、`6379`、`3306`、`11211`、`9200`

## 探测技巧
- `状态码差异`：200/403/500 的变化可用于判断路径存在。
- `长度差异`：错误模板一致时看 body 长度。
- `时间差`：连接超时 vs 连接拒绝。

## 常见内网服务指纹
- Redis：访问 6379 常返回协议错误或无 HTTP 格式响应。
- MySQL：3306 可能返回握手包特征（在半盲中常体现为“有响应但格式异常”）。
- Elasticsearch：9200 常见 JSON banner。
- Docker API：`2375` 可能无鉴权。

## 风险提示
- 扫描范围别上来就全段，先小范围高价值目标。
- CTF 可激进；真实环境应合法授权并控制请求频率。
