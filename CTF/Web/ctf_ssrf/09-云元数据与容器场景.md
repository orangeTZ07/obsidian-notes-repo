# 云元数据与容器场景

## 为什么先看 169.254.169.254
这是链路本地地址，常用于云实例元数据服务。
拿到凭据后，SSRF 危害会从“内网探测”升级为“云资源接管”。

## AWS 常见路径（示意）
- `http://169.254.169.254/latest/meta-data/`
- `http://169.254.169.254/latest/meta-data/iam/security-credentials/`

提示：新版本通常需要 `IMDSv2 token`，题目会简化但要知道真实限制。

## 阿里云/Tencent/GCP/Azure
不同云厂商路径不同，但思路一致：
1. 探测元数据根路径是否可达。
2. 枚举实例信息、角色信息、临时密钥。
3. 回收证据并尝试云 API 横向。

## 容器/K8s 关联目标
- `kubelet`、`etcd`、内部管理 API
- Pod 内服务账号 token 挂载路径（若可 file 读取）

## CTF 实战建议
- 先确定是否云环境题，不要盲目爆扫。
- 出现 `metadata`、`iam`、`role` 等关键词时优先切云链路。
