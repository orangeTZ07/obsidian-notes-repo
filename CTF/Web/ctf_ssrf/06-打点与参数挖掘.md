# 打点与参数挖掘

## 高价值参数名
- `url`
- `uri`
- `path`
- `target`
- `image`
- `callback`
- `next`
- `return_url`
- `feed`
- `proxy`

## 高价值业务功能
- 远程图片上传 / 下载
- 网站预览与缩略图
- webhook 测试
- 报表导入 URL
- 在线抓取、翻译、解析器

## 打点步骤
1. Burp 抓包，标记所有“看起来像地址”的参数。
2. 替换为你可观测域名（DNSLog/Webhook）验证是否出网。
3. 若有出网，逐步替换为内网地址并记录差异。
4. 比较响应长度、状态、耗时，建立“端口/路径存在性”判断。

## 快速验证 payload
- `http://<你的域名>/ssrf-test`
- `http://127.0.0.1/`
- `http://127.0.0.1:80/`
- `http://127.0.0.1:6379/`
- `http://169.254.169.254/`

## 常见误判
- 误把前端请求当 SSRF（其实是浏览器自己请求）。
- 误把后端缓存命中当“目标可达”。
- 误把 WAF 统一错误页当“目标不存在”。
