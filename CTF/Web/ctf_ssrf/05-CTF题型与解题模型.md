# CTF 题型与解题模型

## 题型 A：回显型 SSRF
特征：请求结果直接显示。

解法：
1. 验证可控：访问 `http://your-collaborator`。
2. 打本机：`127.0.0.1`、`localhost`。
3. 扫常见路径：`/flag`、`/admin`、`/debug`。
4. 扫端口和内网段（小范围优先）。

## 题型 B：Blind SSRF
特征：页面几乎不变。

解法：
1. 用 DNSLog 验证请求是否发出。
2. 利用时间差（请求慢接口）判断端口状态。
3. 外带关键内容（如果支持拼接）。

## 题型 C：过滤绕过
特征：禁 localhost、禁 http、禁关键字。

解法：
1. 换 IP 写法（十进制/八进制/IPv6）。
2. 编码绕过（一次/二次编码）。
3. 利用重定向中转。
4. 协议替代（gopher/file/dict 视环境）。

## 题型 D：云环境 SSRF
特征：考元数据接口。

解法：
1. 先测 `http://169.254.169.254/`。
2. 按云厂商路径枚举（见 [[09-云元数据与容器场景]]）。
3. 拿 token / role / 临时凭据。

## 题型 E：协议进阶链
特征：需要 gopher 打 Redis/MySQL。

解法：
1. 确认可用 gopher。
2. 构造原始 payload 并 URL 编码。
3. 写入计划：先探活再执行关键动作。

## 通用解题模型（建议背熟）
`识别输入 -> 验证外连 -> 定义反馈通道 -> 绕过限制 -> 目标优先级攻击 -> 证据回收`
